## 用語集 & 補足

### 1. オペランド

- オペランド：演算子によって計算や操作の対象となるデータ・変数・値のこと
  - 主に変数や値がオペランドに相当する
  - 例：`b = a + 5` の場合、 a, b, 5がオペランド
- 演算子：数値の計算・値の比較などに使用される記号やキーワードのこと
    - `b = a + 5` の場合、=と+が演算子
- リテラル：ソースコードに記載されている具体的な値そのもの
  - 例：5・"Hello World"など
  - リテラルはデータ型を持っている
- エスケープシーケンス：String型やchar型で使われる特殊文字
  - Macの場合は「¥」のかわりに「\」(\optionキー＋¥)を使用する 
- テキストブロック：複数行にわたる文字列を直感的にかつ簡潔に記述ための方法
  - 2つの"""とその挟まれた各行のうち、最も左側に書いた文字を記述した部分を、複数行リテラルの左側とみなす。 

### 2. 式の評価

- 式の評価：Java が式に従って計算処理すること  
以下の原則に基づいて式の評価が行われる

#### 原則1：評価結果への置換の原則

演算子は周辺のオペランドの情報を使って計算を行い、  
それらのオペランドを巻き込んで新しい結果に置き換わる。

- 例1：1 + 5  
→ この計算結果が6に化ける。  


- 例2：1 + 5 - 3  
→ 左側の1 + 5が評価されて6に化けて6 - 3となる。  
→ この計算結果が3に化ける。 

#### 原則2：優先順位の原則

式に演算子が複数ある場合は、Java で定められた優先順位が高い演算子から順に評価される。

- 例1：1 + 5 * 3  
→ 5 * 3の方が優先順位が高いため、1 + 15に化ける  
→ この計算結果が16に化ける。


- 例2：(1 + 5) * 3  
→ ()の方が優先順位が高いため、6 * 3に化ける  
→ この計算結果が18に化ける。

#### 原則3：結合規則の原則

式の中に同じ優先順位グループに属する演算子が複数ある場合、  
演算子ごとに決められた方向から順番に評価される。

- 例1： 10 + 5 + 2  
→ 最初に10 + 5が評価されて15に化けて15 + 2となる  
→ この計算結果が17に化ける。


- 例2：a = b = 10  
→ 最初にb = 10が評価され、bに10が代入される。その結果、変数bの値は10となり、a = 10 となる。  
→ a = 10が評価され、aに10が代入される。その結果、変数aの値は10となる。

### 3. 演算子

#### 算術演算子

| 演算子 | 機能        | 優先順位 | 評価方向 | 評価の例                       |
|-----|-----------|------|------|----------------------------|
| +   | 加算        | 中    | 左から右 | 3 + 5 → 8                  |
| -   | 減算        | 中    | 左から右 | 10 - 3 → 7                 |
| *   | 乗算        | 高    | 左から右 | 3 * 2 → 6                  |
| /   | 除算        | 高    | 左から右 | 3.2 /2 → 1.6<br/>9 / 2 → 4 |
| %   | 剰余（除算の余り） | 高    | 左から右 | 9 % 2 → 1                  |

純粋に9 / 2を行うと、両方とも型がintのため計算結果が4となる。  
計算結果を4.5にしたい場合は9.0 / 2 のように片方をdouble型にする必要がある。

#### 文字列結合演算子

| 演算子 | 機能     | 優先順位 | 評価方向 | 評価の例                                                      |
|-----|--------|------|------|-----------------------------------------------------------|
| +   | 文字列の連結 | 中    | 左から右 | "Hello " + "World" → "Hello World"<br/>"ベスト" + 3 → "ベスト3" |

#### 代入演算子

| 演算子 | 機能       | 優先順位 | 評価方向 | 評価の例              |
|-----|----------|------|------|-------------------|
| =   | 右辺を左辺に代入 | 最低   | 右から左 | a = 10 → a(中身は10) |

#### 複合代入演算子

| 演算子 | 機能                   | 優先順位 | 評価方向 | 評価の例                          |
|-----|----------------------|------|------|-------------------------------|
| +=  | 左辺と右辺を加算して左辺に代入      | 最低   | 右から左 | a += 2 → a (a = a + 2と同じ)     |
| -=  | 左辺と右辺を減算して左辺に代入      | 最低   | 右から左 | a -= 2 → a (a = a - 2と同じ)     |
| *=  | 左辺と右辺を乗算して左辺に代入      | 最低   | 右から左 | a *= 2 → a (a = a * 2と同じ）     |
| /=  | 左辺と右辺を除算して左辺に代入      | 最低   | 右から左 | a /= 2 → a (a = a / 2と同じ）     |
| %=  | 左辺と右辺を除算し、その余りを左辺に代入 | 最低   | 右から左 | a %= 2 → a (a = a % 2と同じ）     |
| +=  | 左辺の後に右辺を連結して代入       | 最低   | 右から左 | a += "風" → a (a = a + "風"と同じ) |

#### インクリメント

| 演算子 | 機能     | 優先順位 | 評価方向 | 評価の例                              |
|-----|--------|------|------|-----------------------------------|
| ++  | 値を1増やす | 最高   | 左から右 | a++ → a<br/>(a = a + 1やa += 1と同じ) |
| --  | 値を1減らす | 最高   | 左から右 | a-- → a<br/>(a = a - 1やa -= 1と同じ) |

++aのようにオペランドの前にインクリメントをつけることもできる。

注意：インクリメントは他の演算子とは一緒に使わないこと！

以下サンプルコード (chapter02/code02-05/src/Main.java)
```
public class Main{
    public static void main(String[] args){
        int a = 10;
        int b = 10;
        System.out.println(++a + 50);   // ①
        System.out.println(b++ + 50);   // ②
    }
}
```
出力結果
```
61
60
```

①の場合と②の場合で出力結果が変わってしまう。

①の場合：  
- 変数aの値が1増える
- 上記に50を加算したものが画面に出力される → 61が出力される

②の場合：
- 変数ｂに50を加算したものが画面に出力される → 60が出力される
- 変数bの値が1増える

このように画面に出力される値が異なり、バグの原因になるため、  
インクリメントは単独で使う。

### 4. 型変換

#### 代入時の自動型変換

小さな型の値を大きな型に代入する場合、  
代入する値を代入先の変数の型に自動的に変換してから代入が行われる。

数値型を小さい順に並べ替えると

- byte
- short
- int
- long
- float
- double

小さい型の値を大きい型に代入する分には、  
代入時に自動型変換が行われ、問題なく動作する。

逆に大きい型の値を小さい型に代入するとコンパイルエラーが発生するため、  
次の強制的な型変換が必要となる。

#### 強制的な型変換

- キャスト演算子：強制的に型変換を行う際にリテラルの前に置かれる演算子。

キャスト演算子の使い方
```
(変換先の型名) 式
```

以下例 (chapter02/code02-07/src/Main.java)
```
public class Main {
    public static void main(String[] args) {
        int a = (int)3.2;   // aには3が代入される
        System.out.println(a);  // 3が出力される
    }
}
```
a = 23なら直接代入できる。  
a = 3.2は直接は代入できない。  
a = (int)3.2だと、一度3.2を3にキャストされてから、a = 3として代入している。

#### 演算時の自動型変換

- 5 / 2 → 2
- 5.0 / 2.0 → 2.5

では異なる型同士の計算の場合はどうなるか？

- 5.0 / 2 → 5.0 / 2.0 → 2.5

意味的に大きな型に統一されてから演算が行われる。

#### 文字列を含む演算時の型変換

片方のオペランドがString型の場合は、  
もう片方のオペランドはString型に変換してから連結する。

以下サンプルコード
```
public class Main{
    public static void main(String[] args){
        String msg = "私の年齢は" + 40;
        System.out.println(msg); 
    }
}
```

出力結果
```
私の年齢は40
```

### 5. 命令実行の文

ここでは主によく使われるものだけをピックアップする。

- 改行せずに出力する：`System.out.print(msg)`
- 2つの値を比較して大きい方を代入する：`int m = Math.max(a,b)`
- 文字列を整数に変換する：`int n = Integer.parseInt(s)`
  - s は数値として解釈できる文字列に限定される
- 乱数を発生させる：`int r = new java.util.Random().nextInt(max)`
  - maxは発生させる乱数の上限値
  - 基本的にimport java.util.Random;をインポートして使う（後述）
- キーボードから1行の文字列の入力を受け付ける：`String s = new java.util.Scanner(System.in).nextLine();`
  - paizaDランク問題では頻出
  - これもimport java.util.Scanner;をインポートして使う(後述)
- キーボードから1つの整数の入力を受け付ける：`int input = new java.util.Scanner(System.in).nextInt();`
  - これもpaizaDランク問題では頻出
  - これもimport java.util.Scanner;をインポートして使う(後述)

Randomをimportで使う場合(chapter02/code02-13/src/Main.java)
```
import java.util.Random;

public class Main{
    public static void main(String[] args) {
        Random random = new Random();
        int r = random.nextInt(90);   // 0〜89までの整数をランダムでrに代入される
        System.out.println(r);
    }
}
```

ScannerをImportで使う場合(chapter02/code02-14/src/Main.java)
```
import java.util.Scanner;

public class Main{
    public static void main(String[] args) {
        System.out.println("あなたの名前を入力してください。");
        Scanner sc = new Scanner(System.in);
        String name = sc.nextLine();  // キーボードから1行分文字列を受け付ける
        System.out.println("あなたの年齢を入力してください。");
        int age = sc.nextInt();   // キーボードから1つ分整数を受け付ける
        System.out.println("ようこそ、" + age + "歳の" + name + "さん");
    }
}
```

## まとめ

- 式
  - 式は演算子とオペランドで構成されている。
  - 演算子が評価されると、その演算子とオペランドは結果に化ける。
  - 演算子は優先順位と結合規則に従って評価される。

- リテラル
  - リテラルにも型があり、記述方法で型は決まる。
  - 文字列リテラル内に改行や引用符などを表現したい場合、エスケープを用いる。
  - テキストブロックを使うと、複数行にわたる文字列情報を直感的に表記できる。

- 型変換
  - 大きい型の変数に小さな型のデータを代入すると、自動的に型が変換され代入される。（代入時の型変換）
  - キャストは小さな型の変数に大きな型のデータを強制的に代入するが、情報の欠損が発生する（強制的な型変換）
  - 式が評価されるとき、大きな型のデータに合わせて自動的に型が変換される（演算時の自動型変換）

- 命令の実行
  - Javaに用意されている様々な命令を実行できる。
