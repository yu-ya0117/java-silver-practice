## 用語集 & 補足

### 1. 開発の流れ

- ソースコード：Javaの文法で書かれた人が読める状態のプログラム
- コンパイル：プログラムをコンピュータが理解できる言語に変換する処理
- Javaの場合は、ソースファイルをクラスファイルに変換する
- コンパイラ：Javaのソースコードをコンパイルするのに使用されるソフトウェアのこと
  - ソースコードの文法のチェックも行われる
- バイトコード：ソースコードのプログラムをJavaの仮想マシンが実行できる状態にした中間言語のこと
- クラスファイル：Javaの仮想マシンで実行するためにバイトコードで書かれたファイル
- インタプリタ：Javaのバイトコードを1行ずつ読み込んで解釈し、コンピュータ語に翻訳しながら実行するプログラム
- JVM：Java Virtual Machineの略、Java仮想マシン。Javaなどのプログラムを実行するために必要なソフトウェア。

### 2. Javaプログラムの基本構造
- ブロック：プログラムの{・・・}で囲まれた部分のこと。
  - 外側のブロックはクラスブロック、内側のブロックはメソッドブロック
- mainメソッド：プログラムを実行した際に一番最初に呼び出される開始地点(エントリーポイント)
  - public static void main(String[] args){}のブロックから処理が始まる
- クラス名：public classの直後についている名前
  - ソースコードのファイル名はクラス名.javaでなければならない。
  - 必ず大文字アルファベットから始まる
  - 例：MainクラスならMain.java

### 3.ソースコードの記述に意識すること

1. 正確に記述する

   - 英数字は半角で入力する
   - 大文字と小文字の違いを区別する
   - o(オー)と0(ゼロ)、l(エル)と数字の1(イチ)、;(セミコロン)と:(コロン)、.(ピリオド)と,(カンマ)を間違えない。
   - カッコ（(),{},[]）や引用符('や")の種類を正確に区別する。

2. 「上から下へ」ではなく、「外から内」へ

   - クラスブロックを書いてすぐ閉じる 
   - 中に入り、メソッドブロックを書いてすぐ閉じる 
   - 中身を一生懸命描く

3. 読みやすいソースコードを記述する

   - インデント：適切な場所で字下げを行う
     - 特にブロックの開始と終了で正確に字下げを行い、カッコの対応とブロックの多重構造の見通しをよくする
   - コメント：プログラムを読みやすくする解説文
     - プログラムのコンパイルと実行では無視される
     - 単数行のコメントと複数行のコメントが存在する

単数行のコメントの場合
```
// コメント：コンパイル・実行は無視される
```

複数行のコメントの場合
```
/*  コメント例：
    コンパイル・実行は無視される  */
```

### 4. 変数宣言

1. 変数宣言の文

```
型 変数名;
```

2. 変数命名の注意点とポイント

- 禁止されている名前（予約語）は使えない
  - int、public、void、staticなど約50種 
- すでに利用している変数名は使えない
- 大文字/小文字、全角/半角は区別される
- 小文字で始まるわかりやすい名前をつける
    - 例：myAgeなど

3. 変数の初期化

```
型 変数名 = 代入するデータ;
```

### 5. データ型

- 整数を格納できる型：byte、short、int、long
  - byte：-128〜127
  - short：-32768〜32767
  - int：-2147483648〜2147483647
  - Long：-9223372036854775808〜9223372036854775807
  - 大体の場合は int が使われる

- 小数を格納できる型：float、double
  - 大体の場合は double が使われる

- 真偽値を格納できる型：boolean
  - 格納できるのは true か false 

- 文字を格納する型
  - 1文字のみ：char
  - String は文字列を扱うクラス
  - String は参照型クラス

### 6. 定数

プログラム中に書き換わることのない値のこと。
以下のように final付きで宣言される。

```
final 型 定数名 = 初期値;
```

一般的に、定数名は全て大文字を用いる。
初期値が代入された後は値の書き換えができない。

## まとめ

- 開発と実行の流れ
  - Javaの文法に従ってソースコードを作成する。
  - ソースコードをコンパイラでコンパイルして、バイトコードに変換する。
  - インタプリタはバイトコードをマシン語（機械語）に変換しながらCPUを動かす。
- 開発の流れと基本構造
  - ソースコードはブロックによる二重構造を持っている。
  - 外部部分は形式記述であり、内側に文を並べて書く。
  - 読みやすいコードを書くには、コメントとインデントを活用する。
- 変数宣言の文
  - 変数は変数名と型を宣言して利用する。
  - 変数名は基本的に自由だが、一定の制約がある。
  - 変数には代表的な9つの型があり、用途に合わせて使い分ける。
  - final をつけて宣言された変数は定数として扱われ、値は書き換えられない。

### 練習1-1
Java でプログラムを開発するためには、「コンパイラ」と「インタプリタ」というソフトウェアが必要です。
「コンパイラ」は、私たちがJavaの文法に沿って記述した「ソースコード」を「バイトコード」に変換してくれます。
「インタプリタ」は内部に持っている「JVM」のしくみを使ってこれを解釈し、マシン語（機械語）に変換してCPUに送り、CPUは命令を実行します。

よって
ア：コンパイラ
イ：インタプリタ
ウ：ソースコード
エ：バイトコード
オ：JVM
